# 瑞幸 MCP 使用说明

## 凭证

瑞幸 Token 来自瑞幸开放平台，绑定瑞幸账号会话。机器人内部支持三类作用域：

- 个人：按飞书用户 `open_id` 保存和使用。
- 群聊默认：按飞书群聊 `chat_id` 保存和使用，仅用于群聊上下文。
- 系统默认：通过配置提供，作为兜底凭证。

## Token 绑定引导

当用户没有可用凭证时，工具不会直接报错，而是发送一张「绑定瑞幸账号」卡片：

- 卡片说明去 [瑞幸开放平台](https://open.lkcoffee.com) 登录复制 Token。
- 卡片内含 Token 输入框；群聊场景额外提供作用域选择（仅个人 / 本群默认），私聊默认存个人。
- 提交后 Token 加密入库，仅展示尾号。
- 支持解绑与查看当前作用域，均不显示完整 Token。

## 点单动线（卡片驱动）

1. 用户说「帮我点一杯生椰拿铁，我在上海人民广场」。
2. 模型调用 `luckin_shop_search`（传 `locationText` 地点描述，可选 `deptName` 关键词）。后端用高德地理编码把地点转成经纬度（GCJ-02），再调瑞幸 `queryShopList`，回门店选择卡片（展示门店名/地址/距离/营业时间/标签），用户点选门店。
3. 选定门店写入会话态（30 分钟），卡片原地变为「已选门店 + 商品搜索框」，后续无需重复选店。
4. 用户在卡内输入商品关键词搜索（异步），机器人回商品选择卡片（左图右文，展示图片/到手价/划线原价/标签）。
5. 用户点选商品：若该商品有规格（杯型/温度/糖度等），卡片变为规格选择卡，选完调 `switchProduct` 刷新价格；否则直接预览并生成「订单确认卡片」。
6. 发起人点击确认后，后端才调用瑞幸 `createOrder`。确认后只创建订单，不会自动支付。

机器人不会让模型直接调用瑞幸 `createOrder`，下单始终需要确认卡片回调触发。用户全程不接触经纬度——只描述地点，由地理编码解析。

## 下单成功与支付

下单成功卡片解析瑞幸 `createOrder` 返回：

- 展示实付金额、订单号。
- 把支付二维码 `payOrderQrCodeUrl` 上传飞书后以图片展示，可直接扫码。
- 微信支付链接 `payOrderUrl` 做成「去微信支付」按钮。
- 附「查看订单状态」按钮，可随时实时刷新订单状态卡。

## 订单生命周期轮询

下单成功后，后台 worker（单实例，默认 30s 一轮）轮询 `queryOrderDetailInfo`，按 `orderStatus` 驱动状态机并主动更新/通知：

- 状态码：10 待付款 / 20 下单成功 / 30 制作中 / 60 等待取餐 / 80 已完成 / 100 已取消。
- 每次状态跃迁记录时间戳，并发节点通知卡（下单成功/制作中/等待取餐含取餐码与预计取餐时间/已完成/已取消）。
- 未支付到达提醒阈值（默认 10 分钟）发一次「还未支付」提醒，不逐秒刷新。
- 终止条件（任一即停）：进入终态（已完成/已取消）；未支付超时（默认 15 分钟）；达到全局最大轮询时长（默认 2 小时）兜底；连续查询失败超上限；凭证失效。

## 地理编码（门店定位）

门店搜索需要经纬度，由地点文本经地理编码得到，并按 query 缓存（24 小时）以节省额度：

- 优先用高德 Web 服务（`amap_key`），坐标系 GCJ-02 与瑞幸一致，最准。注意高德个人 key 的地理编码免费额度有限。
- 高德失败或未配置 key 时，自动回退到 OpenStreetMap Nominatim（免费、无 key），返回的 WGS-84 坐标会自动转换为 GCJ-02。
- 两者都失败时，提示用户更换更具体的地点描述。

## 配置

在 `config.toml` 中配置：

```toml
[luckin_mcp]
system_token = ""      # 可选系统默认瑞幸 Token
credentials_key = ""   # 32 字节 token 加密 key，用于保存个人或群聊默认 Token
server_url = ""        # 可选，默认 https://gwmcp.lkcoffee.com/order/user/mcp
amap_key = ""          # 可选，高德 Web 服务 key，用于门店定位地理编码；不配则只用免费 Nominatim 兜底
unpaid_remind_threshold_seconds = 0  # 可选，未支付提醒阈值秒数，默认 600
unpaid_timeout_seconds          = 0  # 可选，未支付超时秒数，默认 900
order_poll_interval_seconds     = 0  # 可选，订单轮询间隔秒数，默认 30
order_poll_max_seconds          = 0  # 可选，订单轮询最大时长秒数，默认 7200
```
